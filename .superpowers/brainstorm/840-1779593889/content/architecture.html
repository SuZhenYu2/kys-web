系统整体架构

金庸武侠 RPG - 实时同步型架构

架构概览
🎮 客户端层 (Web Browser)
游戏地图渲染 (Canvas/WebGL)
角色移动控制
聊天/对话 UI
WebSocket 连接管理
↑ ↓ WebSocket 实时通信
🌐 网关层
负载均衡 (Nginx)
WebSocket 网关
⚙️ 服务端层 (Spring Boot)
玩家会话管理
地图/场景管理
NPC 行为引擎
战斗系统
任务系统
DeepSeek AI 代理
💾 数据层
MySQL (玩家数据)
Redis (实时状态缓存)
DeepSeek API

核心数据流

  1. 玩家移动：客户端 → WebSocket → 服务端验证 → Redis 更新位置 → 广播给视野内玩家
  2. NPC 行为：定时任务驱动 NPC 决策 → 更新位置/状态 → 广播给附近玩家
  3. AI 对话：玩家触发对话 → 收集上下文 → 调用 DeepSeek API → 返回 NPC 回复 → 广播
  4. 玩家交互：战斗/交易/组队请求 → 服务端验证 → 状态变更 → 双方同步
