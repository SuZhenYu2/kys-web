# 《金庸群侠传·江湖行》完整游戏策划文档

## 一、游戏概述

### 1.1 游戏名称
《金庸群侠传·江湖行》（代号：KYS-RPG）

### 1.2 游戏类型
2D 俯视角多人在线武侠 RPG（MMORPG）

### 1.3 核心体验
**"在金庸的武侠世界中，与 AI 驱动的江湖人物共同生活"**

玩家进入一个由 DeepSeek 大模型驱动的动态武侠世界，NPC 不再是冰冷的任务发布机，而是有记忆、有情感、有自主行为的"江湖居民"。玩家可以拜师学艺、结交朋友、创立门派、参与江湖恩怨，与其他玩家和 AI NPC 共同书写属于自己的武侠传奇。

### 1.4 目标平台
Web 浏览器（PC + 移动端适配）

### 1.5 技术栈
- **前端**：HTML5 Canvas + WebSocket
- **后端**：Java Spring Boot 4 + WebSocket (STOMP)
- **数据库**：MySQL 8.0 + Redis 7.0
- **AI 引擎**：DeepSeek API
- **部署**：Docker + Nginx

| **福缘** | 暴击/奇遇 | 暴击率、随机事件 |

1. **选择性别**：男/女
2. **选择出身**：
   - 孤儿（悟性+5，福缘+3）

4. **命名**：2-6 个汉字

### 3.2 武功系统

#### 3.2.1 武功分类

| 类型 | 说明 | 代表武功 |
|------|------|----------|
| **拳掌** | 近身外功 | 降龙十八掌、太极拳 |

### 3.3 战斗系统

#### 3.3.1 战斗模式

1. **即时战斗**：野外遇敌自动进入战斗
2. **回合制**：PVP、副本采用回合制
3. **切磋**：友好比试，不会死亡

#### 3.3.2 战斗规则

- **出手顺序**：身法决定
- **攻击类型**：普通攻击 / 武功招式 / 暗器 / 道具
- **防御方式**：格挡、闪避、招架
- **状态效果**：中毒、内伤、点穴、眩晕
- **逃跑**：身法差距决定成功率

#### 3.3.3 死亡惩罚

- 掉落 10% 银两
- 随机掉落一件非绑定装备
- 回城复活，气血恢复 10%
- 可选择原地复活（消耗还魂丹）

### 3.4 门派系统

- 找到门派 NPC 申请加入
- 通过入门考验（战斗/答题/寻物）
- 成为外门弟子 → 内门弟子 → 核心弟子 → 长老 → 掌门

#### 3.4.2 门派贡献

- 参与门派战

#### 3.4.3 门派战

- 获胜门派获得全服 Buff 和专属资源

- **婚姻**：异性角色，举办婚礼，获得夫妻技能、双人坐骑

#### 3.5.4 组队系统

## 四、AI NPC 系统（核心特色）

### 4.1 NPC 设计理念

**"每个 NPC 都是活的江湖人物"**

NPC 不再是固定的任务发布机，而是：
- 有自己的身份背景和性格
- 有日常作息和自主行为
- 能记住与每个玩家的互动历史
- 能同时与多个玩家对话
- NPC 之间也会互动

### 4.2 NPC 类型

#### 4.2.1 按功能分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **宗师** | 传授顶级武功 | 张三丰、风清扬 |
| **商人** | 买卖物品 | 客栈老板、铁匠 |
| **任务** | 发布任务 | 各门派执事 |
| **剧情** | 推动主线 | 郭靖、黄蓉 |
| **野怪** | 战斗对象 | 山贼、猛兽 |
| **平民** | 背景人物 | 村民、渔夫 |

#### 4.2.2 按智能等级分类

| 等级 | 说明 | 数量 |
|------|------|------|
| **传奇** | 金庸小说主角/重要角色，深度 AI | 50+ |
| **精英** | 有完整背景故事，中度 AI | 200+ |
| **普通** | 有基础性格，轻度 AI | 1000+ |
| **背景** | 简单对话，无 AI | 5000+ |

### 4.3 NPC 行为系统

#### 4.3.1 作息系统

每个 NPC 有自己的作息表：

```
06:00 - 起床、晨练
08:00 - 早餐、开门营业
10:00 - 工作/修炼
12:00 - 午餐、午休
14:00 - 工作/会客
18:00 - 晚餐、社交
20:00 - 夜生活/修炼
22:00 - 休息
```

不同身份 NPC 作息不同：
- **商人**：营业时间更长
- **武者**：修炼时间更多
- **夜行侠**：夜间活动

#### 4.3.2 移动系统

- NPC 在固定区域内随机移动
- 有目的移动（去吃饭、去修炼）
- 遇到玩家可能停下对话
- 遇到其他 NPC 可能互动

#### 4.3.3 情绪系统

| 情绪 | 触发条件 | 表现 |
|------|----------|------|
| **高兴** | 收到礼物、完成好事 | 主动打招呼、给予优惠 |
| **愤怒** | 被攻击、被欺骗 | 拒绝服务、呼叫守卫 |
| **悲伤** | 亲友死亡、任务失败 | 对话消极、闭门不出 |
| **警惕** | 发现可疑人物 | 保持距离、暗中观察 |
| **好奇** | 遇到陌生人/奇事 | 主动搭话、询问来历 |

### 4.4 AI 对话系统

#### 4.4.1 对话触发方式

1. **主动接近**：玩家走到 NPC 附近
2. **NPC 主动**：NPC 看到玩家主动搭话
3. **事件触发**：特定事件后 NPC 主动找玩家

#### 4.4.2 对话上下文管理

```
NPC 对话上下文 = {
  NPC 身份信息（姓名、身份、性格）
  当前场景信息（地点、时间、天气）
  周围人物信息（附近玩家和 NPC）
  与该玩家的历史对话摘要
  与该玩家的关系值
  当前游戏世界事件
}
```

#### 4.4.3 DeepSeek API 调用策略

**分层调用策略**：

| 层级 | 触发条件 | 模型 | 响应时间 |
|------|----------|------|----------|
| **缓存层** | 常见问题 | 本地规则 | < 50ms |
| **轻量 AI** | 日常对话 | DeepSeek-lite | < 500ms |
| **完整 AI** | 重要剧情/复杂对话 | DeepSeek-chat | < 2s |

**上下文压缩**：
- 保留最近 10 轮对话
- 历史对话自动摘要
- 关键事件永久记忆

#### 4.4.4 多玩家对话处理

- NPC 同时与多个玩家对话时，能"感知"到所有人的存在
- 玩家 A 说的话，玩家 B 也能看到（如果在附近）
- NPC 会在对话中提及在场的其他玩家
- 玩家之间可以通过 NPC 传话

### 4.5 NPC 记忆系统

#### 4.5.1 记忆层级

| 层级 | 内容 | 存储方式 |
|------|------|----------|
| **瞬时记忆** | 当前场景的人和事 | Redis，5 分钟过期 |
| **短期记忆** | 今天发生的事 | Redis，24 小时过期 |
| **长期记忆** | 重要事件、人物关系 | MySQL，永久存储 |
| **核心记忆** | 身份、性格、价值观 | 系统预设，不可变 |

#### 4.5.2 记忆摘要生成

每天凌晨，系统为每个 NPC 生成"日记摘要"：
```
今日摘要：
- 上午在武馆教了 3 个弟子
- 下午遇到玩家[令狐冲]，切磋了一场，赢了
- 晚上和[岳不群]吵架，因为他偷看了秘籍
- 心情：一般
```

摘要用于后续对话的上下文。

## 五、任务系统

### 5.1 任务类型

| 类型 | 说明 | 奖励 |
|------|------|------|
| **主线** | 推动剧情，必须完成 | 大量经验、关键道具 |
| **支线** | 丰富世界观 | 中等奖励 |
| **日常** | 每日刷新 | 稳定收益 |

### 5.2 奇遇系统

**触发条件**：
- 特定地点 + 特定时间 + 特定条件
- 福缘属性影响触发概率
- 某些奇遇需要前置条件

**奇遇示例**：

| 奇遇名称 | 触发地点 | 条件 | 奖励 |
|----------|----------|------|------|
| **华山论剑** | 华山之巅 | 武功排名前100 | 天下第一称号 |
| **琅嬛福地** | 大理无量山 | 携带段氏血脉 | 凌波微步秘籍 |
| **独孤求败** | 襄阳剑冢 | 剑法熟练度>80 | 玄铁重剑 |
| **珍珑棋局** | 擂鼓山 | 悟性>90 | 北冥神功 |

### 5.3 AI 动态任务

NPC 根据当前状态和世界事件，动态生成任务：

```
示例：
- 黄蓉今天心情不好（因为和郭靖吵架）
- 她可能会发布任务：去摘一朵绝情谷的情花来
- 或者：去教训一下城外闹事的恶霸
- 任务内容和奖励会根据她的情绪变化
```

### 8.2 关键技术方案

#### 8.2.1 实时同步方案

```java
// 视野管理 - 使用空间哈希
public class VisionManager {
    // 地图划分为 100x100 的格子
    // 每个格子维护其中的玩家和 NPC 列表
    // 移动时只通知相邻格子的实体
    
    public void onPlayerMove(Player player, Position newPos) {
        Set<Entity> oldNeighbors = getNeighbors(player.getPosition());
        Set<Entity> newNeighbors = getNeighbors(newPos);
        
        // 离开视野
        for (Entity e : oldNeighbors) {
            if (!newNeighbors.contains(e)) {
                sendLeaveVision(e, player);
            }
        }
        
        // 进入视野
        for (Entity e : newNeighbors) {
            if (!oldNeighbors.contains(e)) {
                sendEnterVision(e, player);
            }
        }
        
        // 更新位置
        updatePosition(player, newPos);
    }
}
```

#### 8.2.2 NPC AI 调度

```java
// NPC 行为调度器
@Component
public class NPCBehaviorScheduler {
    
    @Scheduled(fixedRate = 1000) // 每秒执行
    public void tick() {
        List<NPC> activeNPCs = npcService.getActiveNPCs();
        
        for (NPC npc : activeNPCs) {
            // 根据当前时间判断行为
            Behavior behavior = decideBehavior(npc);
            
            switch (behavior) {
                case MOVE:
                    executeMove(npc);
                    break;
                case INTERACT:
                    executeInteract(npc);
                    break;
                case REST:
                    executeRest(npc);
                    break;
                case AI_DIALOGUE:
                    // 需要调用 DeepSeek
                    executeAIDialogue(npc);
                    break;
            }
        }
    }
}
```

```java
public class DeepSeekService {

    public String generateDialogue(NPC npc, Player player, String playerInput) {
        // 构建上下文
        DialogueContext context = DialogueContext.builder()
            .npcIdentity(npc.getIdentity())
            .npcMood(npc.getCurrentMood())
            .currentScene(getCurrentScene(npc))
            .nearbyEntities(getNearbyEntities(npc))
            .playerHistory(memoryService.getHistory(npc.getId(), player.getId()))
            .relationship(memoryService.getRelationship(npc.getId(), player.getId()))
            .worldEvents(getRecentWorldEvents())

        // 调用 DeepSeek API

    }
}
```
